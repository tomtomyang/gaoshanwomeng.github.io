git 家族式命令
Tomtom Young 发布于2021-06-10 更新于2022-05-17 共786字 阅读2分钟

1. 仓库操作

1. 初始化配置

1git config -global user.name <name> #设置提交者名字
2git config -global user.email <email> #设置提交者邮箱
3git config -global core.editor <editor> #设置默认文本编辑器
4git config -global merge.tool <tool> #设置解决合并冲突时差异分析工具
5git config -list #检查已有的配置信息

2. 创建本地仓库

1git init #初始化本地版本库

3. 关联本地仓库和远程仓库

1git remote add [shortname] [url]
2git remote add origin <url>

4. 克隆远程仓库

1git clone <url> #克隆远程版本库
2// 如 git clone git@192.168.9.19:myproject.git

2. 文件操作

1. 添加

1git add . #添加所有改动过的文件
2git add <file> #添加指定的文件

2. 删除

1git rm <file> #删除文件 强制删除 -f
2git rm -cached <file> #停止跟踪文件但不删除

3. 重命名

1git mv <old> <new> #文件重命名

4. 提交

1git commit -m <file> # 提交指定文件
2git commit -m “commit message” #提交所有更新过的文件

5. 推送

3. 分支操作

1. 查看分支

1git branch  #查看本地分支
2git branch -r  #查看远端分支
3git branch -a  #查看所有分支，包括本地和远程的

2. 创建本地分支

1git branch <user_branch_name>  #在本地创建一个新分支
2git checkout -b <user_branch_name>  #在本地创建一个新分支，并且立即切换到这个分支（第一种方式只是创建了一个分支，并没有切换）

3. 切换本地分支

1git checkout master  #从dev分支，切换到master分支

4. 合并本地分支

1git merge dev  #将dev分支合并至master分支

5. 删除本地分支

1git branch -d <branch_name>  #删除本地branch_name分支

6. 推送本地分支到远端

1#local_branch_name用来指明要推送本地哪个分支
2#remote_branch_name用来指明远端接收这个分支后叫什么。
3git push origin [local_branch_name]:<remote_branch_name> 
4
5#如果希望推送当前分支，那么local_branch_name:可以省略
6git push origin remote_branch_name
7
8#另外本地分支名与远端分支名最好保持一致

其实只有master一个分支时做推送操作的命令中 git push origin master，最后这个master其实就是远端的master分支名，master其实也是一个分支，git要求版本库中必须至少要有一个分支，默认叫master。

7. 删除远程分支

1git branch -r -d <remote_branch_name>  #-r表示操作的是远端  -d表示执行删除操作

删除已被删除的远程分支所对应的本地分支：

1git fetch -p
