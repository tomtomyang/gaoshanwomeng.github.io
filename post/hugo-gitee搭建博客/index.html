📅 2020-05-26 👦 Tomtom Young 📖 1064字 ⏱ 3分钟

hugo官网：https://gohugo.io/

hugo主题：https://themes.gohugo.io/

1.安装hugo

  1. 在hugo的git仓库下载最新版本

    ubuntu系统选择hugo_0.71.1_Linux-64bit.deb

  2. 安装完成后在命令行进行验证

1hugo version
2Hugo Static Site Generator v0.71.1-A301F6B2 linux/amd64 BuildDate: 2020-05-25T09:09:07Z
3如果显示版本信息说明安装成功
  1. 初始化hugo博客，生成blog文件夹，进入blog文件夹
1hugo new site blog
2cd blog
  1. blog文件夹的目录结构如下：
1├─archetypes
2├─content 博客文章存放的文件夹
3├─data
4├─layouts
5├─static 静态资源文件夹，可以放favicon,profile,自定义css等
6├─themes 主题文件夹
7└─config.toml blog的配置文件

2.安装主题

  1. 进入hugo themes页面，选一个喜欢的主题下载
1git clone 主题地址 themes/主题文件夹
  1. 配置config.toml
1vim config.toml
2
3theme="主题文件夹名"
4baseURL="服务器域名地址/"
5languageCode="zh-CN"
  1. 创建一篇测试文章
 1hugo new post/test.md
 2cd content/post
 3typora test.md
 4
 5+++
 6aliases = ["test","blog"]
 7title = "test"
 8author = "Hugo Authors"
 9tags = ["test"]
10+++
  1. 生成静态文件，本地预览
1hugo -D
2会生成public文件夹
3hugo server --buildDrafts

3.部署到服务器

这里以免费的gitee pages为例，github pages，gitlab pages都是一样的操作。

1. 推送到远端仓库

将public文件夹里的全部内容push到gitee仓库

1cd public
2git init
3git add *
4git commit -m "blog first commit"
5git remote add origin 仓库地址
6git push -u origin master
7会要求输入gitee的账号和密码

2. 开启gitee pages服务

3. 访问页面

打开gitee page 生成的地址，就可以看到博客已经部署到服务器上

4. 可能遇到的问题

如果出现页面的css样式没有加载的情况，需要在blog目录下新建一个_config.yml文件进行配置

 1touch __config.yml
 2vim __config.yml
 3
 4添加以下内容
 5theme: npq-hugo
 6
 7hugo -D
 8cd public
 9git add *
10git commit -m "second commit"
11git push -u origin master
12
13打开gitee page配置页面，点击更新
14重新访问博客，css样式加载成功
15

4.自定义博客主题

1. 修改主题文件

修改下载的主题一般在三个地方

 2添加对应的参数，例如：
 3[ params ]
 4title = "Tomtom Young"
 5auther = "Tomtom Young"
 6description = "hahahaha"
 7profilePicture = "images/XXX.jpg"
 8keywords = ""
 9favicon = "favicons/"
10customCss = []
11mainSections = ["post"]
12
13blog/themes/主题/.../css文件
14修改页面样式
15
16blog/themes/主题/.../html文件
17修改页面结构

2. 可能遇到的问题

如果配置的favicon图标在火狐浏览器不生效，需要修改引入favicon的link，在favicon.ico前额外加一条/，可以解决图标不加载问题

1 <link rel="shortcut icon" href=".../favicon.ico" type="image/x-icon">
