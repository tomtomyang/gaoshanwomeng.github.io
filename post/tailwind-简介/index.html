1. CSS 发展历程

CSS 发展到现在，基本经历了三个阶段，如下所示：

1. 原生写法

是类似于编程中面向过程的写法，需要什么样式，自己在 css 中写什么样式。

对代码有洁癖的程序员会进行简单的 css 复用。但是也只是简单的复用，大多数时候还是需要什么写什么，想怎么写怎么写。

2. CSS 组件化

应用了面向对象的写法，将相同视觉的 UI 封装成一个组件。

比如一个按钮，整个项目中，这个按钮被多次使用，并且样式一致。那么就可以封装成一个按钮类，使用的时候直接使用这个类名称就 OK。

这也是 bootstrap，elementUI，Antd 等 UI 框架的做法。这种框架的优势在于，封装了大量常见的 UI，比如你需要一个表单，或者需要一个导航，都有现成的 class，我们可以直接拿过来用，快速的完成效果，基本不需要动手写 CSS。

这也是目前比较流行的方法，当下很少有项目是开发者自己一点一点手写样式的了，多多少少都会使用到一些 UI 框架。对于一些需要快速交付的项目，非常适合使用这种组件化 css 框架。

3. CSS 原子化

从技术的角度，CSS 样式编辑可以划分为四个粒度：

1<div style="{ borderRadius: '0.5rem', padding: '1rem' }"> Click </div>
2<div class="rounded-lg p-4"> Click </div>
3<div class="button"> Click </div>
4<Button> Click </Button>

越往下走，颗粒度越来越大，约束性变高，自由性不足。而 TailwindCSS 位于第二层。

第一个阶段的时候，我们讲了有些有对代码有追求的人，会开始复用 CSS。

比如，页面中大量的用到 float: left，那么就可以封装一个类，比如是这样

1.left {float:left}

然后需要使用 float: left 的时候，直接使用 .left 就可以。

Tailwind CSS 做了什么呢？

它将所有的 css 属性全部封装成语义化的类，比如你想要一个 float: left，它已经帮你封装好了，你直接使用一个 float-left 就可以。需要一个宽度为 12 像素，只需要写 w-3 就可以。

2. Tailwind CSS

1. 为什么要原子化？

上面我们说到，早在多年前，就有过 CSS 原子化的实践方案，为什么到现在，原子化 CSS 重新变成了流行的、合适的、新一代的 CSS 框架？

答案是随着其他技术的发展，CSS 的使用方式和职责发生了变化，原子化 CSS 成为了更合适的技术方案。

再详细的说明就是，随着 react，vue 这些前端工程化工具的流行，组件化变得非常容易也非常必须。组件化要求开发者做出更好的样式分离实践，

对于组件的样式而言，采用嵌套的 css 选择器结构: .body > .header {color: red} 来设定样式，不如直接在模板内使用内联样式：<body> <header style={color: red}></header></body> 更方便维护的。

也就是说我们需要将组件的样式单纯的封装在组件内，而不是再进行全局的定义和选择。

2. Tailwind 的优势

1<div style="{ borderRadius: '0.5rem', padding: '1rem' }"> Click </div>
2<div class="rounded-lg p-4"> Click </div>

对比上面这两种写法，相比于行内样式可以发现 Tailwind CSS 的写法也没有省几个字，并不是特别简单，Tailwind CSS 的特点主要表现在下面几个方面：

1. 简单、方便

对于一个三等分的 Grid 属性来说，一个 grid-cols-3 绝对方便：

1.grid-cols-3    {
2  grid-template-columns: repeat(3, minmax(0, 1fr));
3}

或者你还能记得 box-shadow 各个位置的参数吗？在 tailwind 中，直接用 .shadow 就会很方便：

1.shadow {
2  box-shadow: 0 1px 3px 0 rgba(0, 0, 0, 0.1), 0 1px 2px 0 rgba(0, 0, 0, 0.06);
3}

如果刚开始还无法记忆所有的类名，使用方括号也是十分简单，无需过多记忆：

1<img class="absolute w-[762px] h-[918px] top-[-325px] right-[62px] md:top-[-400px] md:right-[80px]" src="/crazy-background-image.png">
2
3<button class="bg-[#1da1f1]">Share on Twitter</button>
4
5<div class="grid-cols-[1fr,700px,2fr]">
6  <!-- ... -->
7</div>

2. 直接、语义化

例如：我想设计一个较大的字体，那我应该设计多大尺寸、使用什么单位？这种情况下，就可以使用 text-lg。

同样还有:

  1. text-white: 白色的色值是哪个来着?
  2. ring: 我想给这个按钮加一个圈圈？
  3. animate-spin: 怎么做一个动画？

3. 规范、统一

Tailwind 拥有一套专业的 UI 属性值，虽然没有封装任何结构性 UI，但是默认提供的一些属性值都是很专业的。比如颜色：

还有各种内边距外边距，宽高，文字大小行高颜色等等。即使你不懂设计，按照他内置的属性做出来的东西，也不会太差。

另外，有了 text-lg 此类，一个页面上就不会出现几十种参差不同的字体大小。

4. 响应式设计

先来看一个在工作中会遇到的响应式布局问题，常见的 CSS 面试题：

响应式布局，一大堆子元素，在大屏幕三等分，中等屏幕二等分，小屏幕一等分？

1<div class="container">
2  <div class="item"></div>
3  <div class="item"></div>
4  <div class="item"></div>
5  <div class="item"></div>
6  <div class="item"></div>
7  <div class="item"></div>
8</div>

通过 grid 布局很容易实现，但未免繁琐：

 1@media (min-width: 1024px) {
 2  .container {
 3    grid-template-columns: repeat(3,minmax(0,1fr));
 4  }
 5}
 6
 7@media (min-width: 768px) {
 8  .container {
 9    grid-template-columns: repeat(2,minmax(0,1fr));
10  }
11}
12
13.conainer {
14  display: grid;
15  gap: 1rem;
16}

使用 tailwind 只要一行，很高效：

1<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4"></div>

5. 修饰符

上面的响应式设计就使用到了修饰符，其他的修饰符还有如各种伪类、暗黑模式、等设计：

1<div class="focus:ring-2 hover:bg-red-700 dark:bg-gray-800"></div>

3. 使用 Tailwind 让文件体积变大

是的。因为它需要把所有的 css 属性全部都封装一遍，所以 CSS 文件巨大，3M 多。所以不建议在页面内直接引入 Tailwind 原生 css 文件的做法。

Tailwind CSS 官方团队为了解决这个问题，提出了一个方案，在编译的时候引入 PurgeCSS 这个工具，构建的时候，会自动删除掉所有你没用到的 CSS。只保留你用到的 CSS。这样最终打包出来的 CSS 文件极小，一般的项目构建出来的 css 文件,压缩一下甚至不会超过 10K，但是也限制了我们必须在工程化项目里使用 Tailwind。

生产环境部署时，可以通过配置它的 purge 项，来对你的 CSS 进行一次 tree-shake ，如果你用的 class 都是它提供的、或者你自己通过 tailwind.config.js 新增的工具类，那么最终编译后只会保留你用过的这些 class 所对应的 CSS （注意不要使用变量去拼接 class 字符串，否则它就不知道你用了哪些 class 了）

3. 可定制的 Tailwind

根据上面所讲的，原子化 CSS 主要是迎合组件化趋势，满足组件内联样式的要求，但是其实此类需求，用 CSS 组件化 UI 框架，也都可以实现，那么对比 ElementUI、AntD 这类框架，Tailwind 如何自处？

用一句话来概括就是，Tailwind 是零散的，举一个例子就是，ElementUI 这类框架，就像是买回来的玩具，除非是动手能力比较强，否则不太能拆开。而 Tailwind 像是积木，需要自己去组装，甚至可以自己生产积木，是高度定制化的。

例如：

在 tailwind.config.js 的 theme 项下新增了一种颜色，取名为 my-green ，实际编写代码时，tailwindcss 就能把我新增的颜色应用到每一个能够设置颜色的属性中，比如图上的背景色 bg-my-green 、文本颜色 text-my-green 、 边框颜色 border-my-green 。

如果一个项目，需要快速或者简单交付，那没必要用 Tailwind CSS。用 bootstrap，bulma 之类的框架可以让你很快的完成一个项目。不用纠结太深的技术。

Tailwind CSS 更适合于页面定制化高的场景。
