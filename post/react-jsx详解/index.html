1. 什么是 JSX

设想如下变量声明：

1const element = <h1>Hello, world!</h1>;

这个语法既不是字符串也不是 HTML，而是React独有的JSX语法。

JSX 全程是 JavaScript XML，是React提供的语法糖，能让我们可以在JS中写html标记语言。

JSX 看似是 HTML 结构，实质是js结构的语法糖，在代码编译阶段被编译成 JS。所以JSX的本质可描述为看似 HTML 结构的 JS 结构。

2. 为什么使用 JSX

下面是 React 官网对于为什么使用JSX的说明：

React 认为渲染逻辑本质上与其他 UI 逻辑内在耦合，比如，在 UI 中需要绑定处理事件、在某些时刻状态发生变化时需要通知到 UI，以及需要在 UI 中展示准备好的数据。

React 并没有采用将标记与逻辑进行分离到不同文件这种人为地分离方式，而是通过将二者共同存放在称之为“组件”的松散耦合单元之中，来实现关注点分离。

React 不强制要求使用 JSX，但是大多数人发现，在 JavaScript 代码中将 JSX 和 UI 放在一起时，会在视觉上有辅助作用。它还可以使 React 显示更多有用的错误和警告消息。

按我个人的理解就是如果不使用 JSX 的语法，那么React组件就是完全使用JS来定义的，直观看起来都是数据和逻辑，而失去了视图的直观性，JSX虽然只是一个语法糖，但是让开发者重新找到了UI视图的直观感，同时还能处理事件绑定和数据绑定，实现了渲染逻辑和数据操作逻辑的松散耦合。

3. JSX 实际做了什么

1. Babel

JSX 在编译时会被 Babel 编译为React.createElement 方法，但是JSX 并不是只能被编译为 React.createElement 方法，你可以通过@babel/plugin-transform-react-jsx插件显式告诉 Babel 编译时需要将 JSX 编译成什么函数的调用（默认为React.createElement）。

1. 安装

Babel中，用于JSX转换的插件是@babel/plugin-transform-react-jsx，因此我们需要安装它，同时要想Babel起作用，我们还需要安装Babel的核心，用于AST的解析转换和生成，还需要Babel-loader：

1npm i @babel/core @babel/plugin-transform-react-jsx babel-loader
