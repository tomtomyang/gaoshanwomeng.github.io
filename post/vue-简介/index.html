1.Vue介绍

1.Vue是什么

  1. Vue.js是优秀的前端 JavaScript 框架

    Vue.js是一套构建用户界面的渐进式框架，Vue只关注视图层， 采用自底向上增量开发的设计，Vue 的目标是通过尽可能简单的 API 实现响应的数据绑定和组合的视图组件。

  2. 库和框架的区别：

    1. 库(如jQuery)

      库是工具，提供大量API，体现了封装的思想、需要自己调用这些API。

    2. 框架

      框架提供了一套完整解决方案，使用者要按照框架所规定的某种规范进行开发 。

2.为什么要学习 Vue

传统开发模式是JQuery+RequireJS/SeaJS+artTemplate+Gulp/Grunt，随着项目业务场景的复杂，传统模式已无法满足需求，就出现了Angular/React/Vue等框架。

目前Vue是排名第三的前端框架，因为他简单易学，很多前端开发工程师可以很快掌握并且应用到实际开发中。

3.Vue 能做什么

  1. 最大程度上解放了 DOM 操作

  2. 单页web项目开发

  3. 传统网站开发

4.核心特性

3.Vue实例的选项

  1. el

    作用:当前Vue实例所管理的html视图

    值:通常是id选择器(或者是一个 HTMLElement 实例)

    不要让el所管理的视图是html或者body!

  2. data

    Vue 实例的数据对象，是响应式数据(数据驱动视图)

    可以通过 vm.$data 访问原始数据对象

    Vue 实例也代理了 data 对象上所有的属性，因此访问 vm.a 等价于访问 vm.$data.a

    视图中绑定的数据必须显式的初始化到 data 中

  3. methods

    其值为可以一个对象

    可以直接通过 VM 实例访问这些方法，或者在指令表达式中使用。

    方法中的 this 自动绑定为 Vue 实例。

    注意不应该使用箭头函数来定义 method 函数 (例如 plus: () => this.a++)。理由是箭头函数绑定了父级作用域的上下文，所以 this 将不会按照期望指向 Vue 实例，this.a 将是 undefined

2.指令

指令 (Directives) 是带有 v- 前缀的特殊特性。

指令的职责是，当表达式的值改变时，将其产生的连带影响，响应式地作用于 DOM。

 1  <div id="app">
 2        <p> {{message}}</p>
 3        <!-- v-on就是vue给标签元素提供的扩展-指令
 4            v-on指令就是给标签绑定事件，这里是click，
 5            当事件处于点击状态时，出发methods中的changeMsg方法
 6        -->
 7        <button v-on:click="changeMsg">按钮</button>
 8    </div>
 9    <!-- 引入vue.js -->
10    <script src="./vue.js"></script>
11    <script>
12        const vm = new Vue({
13            el: '#app'，
14            data: {
15                message: '我是data中的message属性的值'，
16                age: 20
17            }，
18            methods: {
19                changeMsg: function() {
20                    this.message += "啦";
21                }
22            }
23        });
24    </script>

2.v-if和v-show

根据表达式的bool值进行判断是否渲染该元素，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。

 1 <div id="app">

 8    </div>

3.v-on

  1. 作用:使用 v-on 指令绑定 DOM 事件，并在事件被触发时执行一些 JavaScript 代码。

  2. 语法: @事件名.修饰符 = “methods中的方法名”

  3. 注意: $event 可以传形参

 1 <div id="app">
 2        <!-- v-on.xx事件名='当触发xx事件时执行的语句' -->
 3        <!-- 执行一段js语句:可以使用data中的属性 -->
 4        <button v-on:click="count += 1">增加 1</button>
 5        <!-- v-on的简写方法 -->
 6        <button @click="count += 1">增加 1</button>
 7        <!-- 执行一个方法 -->
 8        <button @click="add">增加 1</button>
 9        <!-- 执行一个方法、这种写法可以传形参 -->
10        <button @click="fn1(count)">执行fn1方法</button>
11        <!-- 执行一个方法、这种写法可以传形参，特殊的形参$event -->
12        <button @click="fn2($event)">执行fn2方法</button>
13        <hr>
14        <!-- 和v-for结合使用 -->
15        <button @click="fn3(index)" v-for="(item， index) in items">执行fn3方法</button>
16        <!-- v-on修饰符 如 once: 只执行一次 -->
17        <button @click.once="fn4">只执行一次</button>
18
19        <p>上面的按钮被点击了 {{ count }} 次。</p>
20    </div>
21    <script src="./vue.js"></script>
22    <script>
23        new Vue({
24            el: '#app'，
25            data: {
26                count: 0，
27                items: ['a'， 'b'， 'c']
28            }，
29            methods: {

31                    this.count += 1;
32                }，
33                fn1: function(count) {
34                    console.log(count);
35                    console.log('fn1方法被执行');
36                }，
37                fn2: function(e) {
38                    console.log(e);
39                    console.log('fn2方法被执行');
40                }，
41                fn3: function(index) {
42                    console.log(index);

12.前端路由

1.什么是单页应用

单页应用(single page web application，SPA)，是在一个页面完成所有的业务功能，浏览器一开始会加载必需的HTML、CSS和JavaScript，之后所有的操作都在这张页面完成，这一切都由JavaScript来控制。

2.单页应用优缺点

  1. 优点

    操作体验流畅

    完全的前端组件化

  2. 缺点

    首次加载大量资源(可以只加载所需部分)

    对搜索引擎SEO不友好 -> 服务端渲染

    开发难度相对较高

3.单页应用的实现原理

前后端分离(后端专注于数据、前端专注于交互和可视化)+前端路由

  1. Hash路由

    利用URL上的hash，当hash改变不会引起页面刷新，所以可以利用 hash 值来做单页面应用的路由，并且当 url 的 hash 发生变化的时候，可以触发相应 hashchange 回调函数。

14.过渡和动画

基本用法就是给我们需要动画的标签外面嵌套transition标签 ，并且设置name属性。

Vue 在插入、更新或者移除 DOM 时，提供多种不同方式的应用过渡效果。在 CSS 过渡和动画中自动应用 class，可以配合使用第三方 CSS 动画库，如 Animate.css。

1.CSS 过渡和动画

Vue 提供了 transition 的封装组件，在下列情形中，可以给任何元素和组件添加进入/离开过渡

2v-enter：定义进入过渡的开始状态。
