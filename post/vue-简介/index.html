Vue 和 React 的优点分别是什么？

(vue) => {渐进式}

Vue2.0 中，“渐进式框架”和“自底向上增量开发的设计”这两个概念是什么？

The Progressive Framework by Evan You

1. Vue 是什么

这一点，Vue的官网说的很清楚：

一句话来说，一套用于构建用户界面的渐进式 JavaScript 框架

这句话的核心点有两处，构建用户界面和渐进式框架。那么这分别代表了什么意思呢？

1. 构建用户界面

这个是没什么难理解的，因为Vue作为一个前端框架被设计出来，就是为了解决数据与视图的问题，Vue被设计为MVVM的架构，采用数据与视图双向驱动的理念，帮助开发者快速的完成界面的开发。

2. 渐进式框架

对于渐进式框架的理解，还要结合另外一句话来理解，就是：

与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。

那么所谓渐进式，所谓自底向上逐层应用，到底是什么意思？

这里可以根据一张图，和一段话，来进行理解：

React的主张主要是函数式编程的理念，比如说，你需要知道什么是副作用，什么是纯函数，如何隔离副作用。它的侵入性看似没有Angular那么强，主要因为它是软性侵入。

你当然可以只用React的视图层，但几乎没有人这么用，为什么呢，因为你用了它，就会觉得其他东西都很别扭，于是你要引入Redux，于是你要纠结业务开发过程中每个东西有没有副作用。

Vue可能有些方面是不如React，但它是渐进的，没有强主张，你可以在原有大系统的上面，把一两个组件改用它实现，当jQuery用；也可以整个用它全家桶开发，当Angular用；还可以用它的视图，搭配你自己设计的整个下层用。你可以在底层数据逻辑的地方用OO和设计模式的那套理念，也可以函数式，都可以，它只是个轻量视图而已，只做了自己该做的事，没有做不该做的事，仅此而已。

渐进式的含义，我的理解是：没有多做职责之外的事。

也就是说Vue搭配其生态，可以实现大型项目的开发，对开发流程，业务逻辑的实现进行强约束，同时我们也可以只使用Vue进行数据到视图的转换，只让Vue关注视图的操作，其他部分自己实现。

因此，我们可以灵活的选择我们使用Vue的方式，既可以把它当作一个开发框架，也可以把它当成是一个类似于jQuery的库。你的开发逻辑只取决于你，Vue不会对你强加干预，但是如果你有这样的需求，那Vue也可以提供。
