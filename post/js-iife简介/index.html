1. 场景

下面这个面试题你大概率遇到过：

1for (var i = 0; i < 5; i++) {
2  setTimeout(() => {
3    console.log(i);
4  }, 10);
5}

这里var会声明一个全局变量i。函数会在10秒后异步执行，当函数在执行时，根据作用域链会在全局变量对象中进行查找，而此时循环已经结束，全局的i为5。所以最终结果为5个5。

闭包可以帮我们保护变量不被外部所污染，也可以帮我们保存变量，以供之后作用域链查找并使用。通过自执行函数，我们将i作为局部变量传入进行保护和保存：

1for (var i = 0; i < 5; i++) {
2  (function fn (i) {
3    setTimeout(() => {
4      console.log(i);
5    }, 10);
6  }(i));
7}

这里自执行函数fn在每次循环的时候，都会创建一个全新的执行上下文。各个执行上下文之间不会相互影响，从而保证每次传入的i不会发生变化。

每次fn对应的执行上下文中，函数() => {console.log(i)}的作用域会在创建的时候确定，其值为当前循环中fn的变量对象。

由于函数() => {console.log(i)}的引用地址会在其它的(setTimeout执行时生成的执行上下文)执行上下文中被占用，而它是在fn中被创建的，所以fn的执行上下文并不会被销毁，进而帮我们保存当前循环中对应的i

1// 这是一段伪代码
2window.setTimeout = function(callback,delay) {
3  // callback会在形参赋值时占用`() => {console.log(i)}`的引用地址
4  // callback = AAAFFF111 (假设AAAFFF111为函数`()=> {console.log(i)}`在堆中的地址)
5  // 10秒后调用callback
6  callback()
7}

所以在10毫秒后执行函数时，函数() => {console.log(i)}中的变量i会从其根据作用域链到对应的循环中的fn中获取，而fn中的i正好是每次循环中传入的i。

当然，由于在es6中引入了let，所以我们可以直接使用let来定义变量保证能获取到每次循环中的i:

1for (let i = 0; i < 5; i++) {
2  setTimeout(() => {
3    console.log(i);
4  }, 10);
5}

其实现过程其实和上例中的自执行函数形成闭包，并将对应的变量传入类似，只不过传入变量并赋值的过程JavaScript引擎帮我们处理了。

for循环有一个特别之处，设置循环变量的那部分是一个父作用域，而循环体内部是一个单独的子作用域

执行过程的伪代码如下：

 1{
 2  // 父块作用域
 3  let i = 0;
 4  // 第一次循环
 5  {
 6    // 子块作用域,将i传进来，并进行形参赋值(相当于有形参i的函数自执行)
 7    let i = 0; // 这里`JavaScript`引擎帮我们传入实参，并进行形参赋值
 8    setTimeout(() => console.log(i), 10);
 9  }
10  i++;
11  // 第二次循环
12  {
13    let i = 1;
14    setTimeout(() => console.log(i), 10);
15  }
16  // ...
17}

2. IIFE是什么

立即执行函数表达式（IIFE）

  • 在 JavaScript 中，每一个函数在执行时都会产生一个新的执行环境。
  • 由于在函数中定义的变量和函数只能在内部访问而不能被外部访问。
  • 这提供了一个非常简单的方法来创建私有作用域。
 1// 因为返回的函数有权访问私有变量 `i`
 2
 3function makeCounter() {
 4    // `i` 只能在 `makeCounter`内被访问.
 5    var i = 0;
 6
 7    return function() {
 8      console.log( ++i );
 9    };
10}
11
12// 注意 `counter` 和 `counter2` 都有私有的作用域 `i`.
13
14var counter = makeCounter();
15counter(); // logs: 1
16counter(); // logs: 2
17
18var counter2 = makeCounter();
19counter2(); // logs: 1
20counter2(); // logs: 2
21
22i; // ReferenceError: i 未定义 (只存在 makeCounter 内部)

需要注意1

现在，无论你用 function foo(){} 还是 var foo = function(){} 的方式定义函数，最终都会以一个函数的标识符结尾，可以通过 () 调用函数，像 foo() 。

1// 像这样定义的函数可以在函数名后放置 () 来执行
2var foo = function(){ }
3// 比如 foo(), 因为 foo 只是函数表达式 `function() { /* code */ }`的引用
4
5// ...是不是只在函数表达式之后放置 () 就能执行?
6function(){ }(); // SyntaxError: Unexpected token (

当解析器在全局范围内或在函数中遇到 function 关键字时，它会认为这是函数声明而不是函数表达式。如果你没有明确告诉解析器这是一个表达式，它会认为这是一个匿名的函数声明并抛出意外的语法错误，因为函数声明需要名称。

需要注意2

如果你为一个函数指定了名称并且在立刻在其后边放置了括号，解析器也会抛出错误，但原因不同。虽然在表达式之后放置括号说明这是一个将被执行的函数，但在声明之后放置括号会与前面的语句分离，成为一个分组操作符（可以作为优先提升的方法）。

 1// 现在这个函数声明的语法是正确的，但还是有报错
 2// 表达式后面的括号是非法的， 因为分组运算符必须包含表达式
 3
 4function foo(){ }(); // SyntaxError: Unexpected token )
 5
 6// 如果你在括号内放置了表达式， 没有错误抛出...
 7// 但是函数也不会执行， 因为:
 8
 9function foo(){ }( 1 );
10
11// 它与一个函数声明后面放一个完全无关的表达式是一样的:
12
13function foo(){ }
14
15( 1 );

3. 闭包，作用域、IIFE

作用域

在 JavaScript 中有两种作用域类型：局部作用域、全局作用域

  • JavaScript 拥有函数作用域：每个函数创建一个新的作用域。
  • 作用域决定了这些变量的可访问性（可见性）。
  • 函数内部定义的变量从函数外部是不可访问的（不可见的）。
  • 像函数被函数名调用时参数会被传递一样，立即执行函数表达式时参数同样会被传递。
  • 在一个函数内部定义的函数可以访问外部函数的变量（这种关系被称为闭包）。
  • 一个立即执行函数表达式可以用于封锁函数值并且有效的存储状态。
 1// 以下程序的运行结果和你想象的并不一样
 2// 因为 `i` 的值不会被锁定。相反，当点击每个链接的时候 (循环已经结束), 会显示元素的总数, 因为那才是点击时 `i` 实际的值.
 3
 4var elems = document.getElementsByTagName( 'a' );
 5
 6// addEventListener(event, function, useCapture) 中useCapture
 7// true - 事件在捕获阶段执行 
 8// false- 默认。事件在冒泡阶段执行
 9
10for ( var i = 0; i < elems.length; i++ ) {
11  	elems[ i ].addEventListener( 'click', function(e){
12    	e.preventDefault();
13    	alert( 'I am link #' + i );
14  	}, 'false' );
15}
16
17// 以下程序会按你想象的方式运行,
18// 因为在 IIFE 中, `i` 的值，会作为 `lockedInIndex` 被锁定。 循环结束之后, 尽管 `i` 的值是元素总数, 但是在 IIFE 中`lockedInIndex` 的值是函数表达式调用时传入的(`i`)的值因此当点击链接时, 显示的值是正确的。
19
20var elems = document.getElementsByTagName( 'a' );
21
22for ( var i = 0; i < elems.length; i++ ) {
23
24  (function( lockedInIndex ){
25    elems[ i ].addEventListener( 'click', function(e){
26      e.preventDefault();
27      alert( 'I am link #' + lockedInIndex );
28    }, 'false' );
29
30  })( i );
31
32}
33
34// 还这样使用 IIFE , 只是包含点击处理函数, 并不是整个 `addEventListener` 声明
35// 无论哪种方式，两个示例都使用IIFE, 我发现前面的例子更易读懂
36
37var elems = document.getElementsByTagName( 'a' );
38
39for ( var i = 0; i < elems.length; i++ ) {
40
41  elems[ i ].addEventListener( 'click', (function( lockedInIndex ){
42    return function(e){
43      e.preventDefault();
44      alert( 'I am link #' + lockedInIndex );
45    };
46  })( i ), 'false' );
47}

立即执行函数表达式最好的一方面就是，因为这个匿名函数表达式被立即执行，没有标识符，所以闭包的使用不会污染当前作用域。

4. IIFE和模块化

模块化方法不仅强大而且简单。你可以用更少的代码有效地命名方法和属性，用一种方式组织所有的代码模块，并且可以避免全局变量的污染以及创建私有作用域。

 1// 创建一个立即执行的匿名函数表达式, 然后将它的 *返回值* 赋给一个变量。这种方法无须再创建一个 `makeWhatever` 函数的引用。
 2
 3// 如同上面 "关于括号的注意事项" 中提到的一样， 尽管括号在函数表达式中不是必须添加的, 但是按照习惯还是应该添加括号.
 4// 因为这可以更清晰的表示出赋值给一个变量的是函数的 *结果* 而不是函数自身
 5
 6var counter = (function(){
 7  var i = 0;
 8
 9  return {
10    get: function(){
11      return i;
12    },
13    set: function( val ){
14      i = val;
15    },
16    increment: function() {
17      return ++i;
18    }
19  };
20}());
21
22// `counter` 是一个有属性的对象, 它的属性都是方法
23
24counter.get(); // 0
25counter.set( 3 );
26counter.increment(); // 4
27counter.increment(); // 5
28
29counter.i; // undefined (`i` 不是返回对象的属性)
30i; // ReferenceError: i 未定义 (它只存在于闭包内)
