学习Javascript闭包（Closure）

在 JavaScript 中闭包无所不在，你必须识别并接纳它。

闭包是词法作用域导致的结果。闭包在你的代码中一直在被创建和使用，你在毫无意识的状态下就使用了闭包。

我们现在要做的事补充我们缺少的思维，来识别，接纳，并以自己的意志利用闭包。

1. 定义闭包

闭包是和词法作用域的概念紧密的结合的在一起的，了解闭包之前必须了解词法作用域。简单来说词法作用域就是执行上下文，里面包含了当前作用域下声明的变量集合。

一个函数和对其周围状态（词法作用域）捆绑在一起（或者说函数被作用域包围），这样的组合就是闭包（closure）。也就是说，闭包让你可以在一个内层函数中访问到其外层函数的作用域。

在 JavaScript 中，每当创建一个函数，闭包就会在函数创建的同时被创建出来。

2. 闭包解决的问题

 1function init() {
 2    var name = "Mozilla"; 
 3    function displayName() {
 4        alert(name); 
 5    }
 6    displayName();
 7}
 8init();
 9// console output
10// Mozilla

在上面的代码中，引擎执行alert(name)时，会进行RHS，在当前作用域中查找name，没有找到，则沿着作用域链，向上一层init的作用域查找，找到了name的值为Mozilla，则交给alert执行。

这是我们已经知道的知识，是作用域和作用域链、LHS和RHS的工作方式。那么换一种情况：

 1function makeFunc() {
 2    var name = "Mozilla";
 3    function displayName() {
 4        alert(name);
 5    }
 6    return displayName;
 7}
 8
 9var myFunc = makeFunc();
10myFunc();
11// console output
12// Mozilla

这里是将makeFunc函数执行的返回值赋值给myFunc，也就是displayName，那么在执行myFunc时，实际执行的是displayName。

引擎进入displayName中执行alert(name)，在函数作用域中查找name，没有找到，那么应该去外层作用域查找，但是这就是问题关键，此时displayName已经被return出来了，存入了myFunc中，那么上一层作用域应该是哪个呢？是dispalyName外面的makeFunc的函数作用域，还是myFunc外面的全局作用域？

要解决这种场景产生的问题，就需要一个机制来告诉我们或者引擎，外层作用域是哪个？这就是闭包的作用。

从实际的代码运行情况看，很明显外层的作用于是dispalyName外面的makeFunc的函数作用域，因为在全局作用于下是没有name的，会输出空。

这是不是就说明，displayName是作为一个闭包，携带着外层的作用域环境，被return出来的，也就是MDN给闭包的定义：

一个函数和对其周围状态（词法作用域）捆绑在一起（或者说函数被作用域包围），这样的组合就是闭包（closure）。也就是说，闭包让你可以在一个内层函数中访问到其外层函数的作用域。

在 JavaScript 中，每当创建一个函数，闭包就会在函数创建的同时被创建出来。

那么我们用闭包的理念来分析另一段代码：

 1function makeAdder(x) {
 2  return function(y) {
 3    return x + y;
 4  };
 5}
 6
 7var add5 = makeAdder(5);
 8var add10 = makeAdder(10);
 9
10console.log(add5(2));
11console.log(add10(2));

首先看赋值部分：

  1. add5的值为makeAdder(5)的返回值，makeAdder(5)的返回值是一个闭包，包括了一个匿名函数已经函数外层的作用域环境，在这个环境中x=5；
  2. add10的值为makeAdder(10)的返回值，makeAdder(10)的返回值是一个闭包，包括了一个匿名函数已经函数外层的作用域环境，在这个环境中x=10；

再看执行部分：

  1. add5(2)的入参为2，进入function(y){ return x + y }内部，执行return，查找x和y的值，x在当前作用域中没有找到，找上层作用域，在闭包中的上层作用域中x = 5，y在当前作用域中存在，y = 2，因此返回值为5 + 2 = 7；
  2. add10(2)的入参为2，进入function(y){ return x + y }内部，执行return，查找x和y的值，x在当前作用域中没有找到，找上层作用域，在闭包中的上层作用域中x = 10，y在当前作用域中存在，y = 2，因此返回值为10 + 2 = 12；

到这里我们已经可以体会到闭包的一些功能了，但是有一个问题，MDN定义描述的有些模糊，就是闭包里保存的作用域，是单层作用域还是一个作用域链，我们再通过一段代码来分析：

 1function makeFunc() {
 2    var name = 1;
 3    function f1() {
 4        function f2(){
 5            return name;
 6        }
 7        return f2
 8    }
 9    return f1()
10}
11
12var myFunc = makeFunc();
13console.log(myFunc, myFunc());
14// console output
15// f2() 1

通过这个结果，我们可以明确知道了，闭包中存在完整的作用域链。

3. 看到闭包

 1var fn;
 2
 3function foo() {
 4	var a = 2;
 5	function baz() {
 6		console.log( a );
 7	}
 8	fn = baz;
 9}
10
11function bar() {
12	fn();
13}
14
15foo();
16bar(); 
17
18// console output
19// 2

我们来分析这里的代码执行：

  1. 执行foo()，foo函数作用域内，声明并赋值了a = 2和baz()，执行fn = baz，对fn进行LHS查找，当前作用域没有，在全局作用域下找到a，对baz进行RHS查找，在当前作用域中找到，赋值给fn，这里是闭包，带有作用域；
  2. 执行bar()，bar函数作用域内，查找fn，没有找到，在全局作用域中找到了，执行fn()，也就是baz的闭包，打印输出2；

4. 使用闭包

知道了闭包可以解决什么问题，或者说，知道了闭包的机制，那么我们可以利用闭包这种特性，来做点什么呢？

1. 存储变量的临时值

在 Web 开发中，你想要这样做的情况特别常见。

大部分我们所写的 JavaScript 代码都是基于事件的，我们会将这个事件绑定到dom元素上（比如click、change）。我们的代码通常作为回调，为响应事件而执行。

 1function makeSizer(size) {
 2  return function() {
 3    document.body.style.fontSize = size + 'px';
 4  };
 5}
 6
 7var size12 = makeSizer(12);
 8var size14 = makeSizer(14);
 9var size16 = makeSizer(16);
10document.getElementById('size-12').onclick = size12;
11document.getElementById('size-14').onclick = size14;
12document.getElementById('size-16').onclick = size16;

上面这段代码中，我们可以感受到，12、14、16这些形参，这些可以改变的值，都被固定到了函数的闭包中。

3. 模拟类的私有变量

 1function Person(){
 2    var name = 'default';
 3    this.getName:function(){
 4        return name;
 5    }
 6    this,setName:function(value){
 7        name = value;
 8    }
 9}
10console.log(Person.getName())//default
11console.log(Person.setName('mike'))
12console.log(Person.getName())//mike
