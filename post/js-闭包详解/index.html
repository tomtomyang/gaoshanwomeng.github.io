学习Javascript闭包（Closure）

在 JavaScript 中闭包无所不在，你必须识别并接纳它。

闭包是词法作用域导致的结果。闭包在你的代码中一直在被创建和使用，你在毫无意识的状态下就使用了闭包。

我们现在要做的事补充我们缺少的思维，来识别，接纳，并以自己的意志利用闭包。

1. 定义闭包

闭包是和词法作用域的概念紧密的结合的在一起的，了解闭包之前必须了解词法作用域。简单来说词法作用域就是执行上下文，里面包含了当前作用域下声明的变量集合。

一个函数和对其周围状态（词法作用域）捆绑在一起（或者说函数被作用域包围），这样的组合就是闭包（closure）。也就是说，闭包让你可以在一个内层函数中访问到其外层函数的作用域。

在 JavaScript 中，每当创建一个函数，闭包就会在函数创建的同时被创建出来。

 1function init() {
 2    var name = "Mozilla"; 
 3    function displayName() {
 4        alert(name); 
 5    }
 6    displayName();
 7}
 8init();
 9// console output
10// Mozilla

在上面的代码中，引擎执行alert(name)时，会进行RHS，在当前作用域中查找name，没有找到，则沿着作用域链，向上一层init的作用域查找，找到了name的值为Mozilla，则交给alert执行。

这是我们已经知道的知识，是作用域和作用域链、LHS和RHS的工作方式。那么换一种情况：

 1function makeFunc() {
 2    var name = "Mozilla";
 3    function displayName() {
 4        alert(name);
 5    }
 6    return displayName;
 7}
 8
 9var myFunc = makeFunc();
10myFunc();
11// console output
12

这里是将makeFunc函数执行的返回值赋值给myFunc，也就是displayName，那么在执行myFunc时，实际执行的是displayName，
