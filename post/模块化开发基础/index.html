📅 2020-06-15 👦 Tomtom Young 📖 4297字 ⏱ 9分钟

1.传统模式弊端

传统的开发模式就是一个页面对应1套html，js，css。这样就造成了3个问题：

  1. 命名冲突

    多个js文件如果存在同名的变量，则会发生变量覆盖。

    1.js

    1

    const a=1

    2.js

    1

    const a=2

  2. 文件依赖

    各个js文件之间不能相互直接按需引用。必须都在.html文件中进行按顺序的引入。

  3. 代码冗余

    类似的页面之间会用大量重复的代码，降低工作效率。

模块化就是为了解决这一系列问题而出现的。

2.模块化开发

1.什么是模块化

模块化就是将单独的一个功能封装到一个单独的文件中，模块之间是相互独立的，但是可以通过特定的接口进行自己的公开和对其他模块的依赖。

2.模块化优势

  1. 方便代码的复用
  2. 提高开发效率，方便后期维护
  3. 解决了传统开发模式的弊端：命名冲突，文件依赖，代码冗余

3.ES6模块化规范

ES6模块化规范是浏览器端和服务器端通用的模块化规范：

  1. 每一个js文件都是一个独立的模块
  2. 导入模块成员使用import关键字
  3. 暴露模块成员使用export关键字

4.bable

bable在我理解应该是一种js语言的转换器，提高js在各个不同内核浏览器和不同版本浏览器的兼容性。

bable的官方定义：

Babel 是一个工具链，主要用于将 ECMAScript 2015+ 版本的代码转换为向后兼容的 JavaScript 语法，以便能够运行在当前和旧版本的浏览器或其他环境中。下面列出的是 Babel 能为你做的事情：

  1. 语法转换
  2. 通过 Polyfill 方式在目标环境中添加缺失的特性 (通过 @babel/polyfill 模块)
  3. 源码转换 (codemods)
  4. 更多！ (查看这些 视频 获得启发)

3.webpack

我们已经学习了ES6的模块化思想，但是在实际开发中，并不能很好的使用模块化思想，即使使用了bable，增强了高级JS的兼容性，但web开发仍然有很多问题存在。

1.当前web开发困境

  1. 文件依赖关系错综复杂
  2. 静态资源请求效率低
  3. 对模块化的支持不够友好
  4. 浏览器对高级js兼容性差

2.webpack概述

Webpack 是一个前端资源加载/打包工具。它将根据模块的依赖关系进行静态分析，然后将这些模块按照指定的规则生成对应的静态资源。

webpack提供了强大的模块化支持，以及代码压缩，处理JS兼容问题，性能优化等一系列功能，极大的提高了开发效率和项目的可维护性。

webpack的官方定义：

本质上，webpack 是一个现代 JavaScript 应用程序的静态模块打包器(module bundler)。当 webpack 处理应用程序时，它会递归地构建一个依赖关系图(dependency graph)，其中包含应用程序需要的每个模块，然后将所有这些模块打包成一个或多个 bundle。

5.webpack配置

1.配置入口与出口

在上面的小例子里，可以得知webpack的默认入口文件为src\index.js，默认出口文件为dist\main.js，如果要修改打包的入口与出口，需要在webpack.config.js中进行配置：

webpack.config.js

 1
 2
 3
 4
 5
 6
 7
 8

10

const path = require('path')

module.exports = {
    mode: 'development',
    entry: path.join(__dirname, './src/index.js'),
    output: {
        path: path.join(__dirname, './dist'),
        filename: 'bundle.js'
    }
}

这里我们指定入口文件是index.js，出口文件为bundle.js。

2.自动打包功能

对于index.js文件，如果我们在打包后再次更改里面的内容，需要再次打包，才能生效，因此需要配置自动打包功能。

  1. 安装dev-server

    dev-server是一个本地服务器插件。

    1

    npm i webpack-dev-server -D

  2. 修改package.json

  3. 修改index.html

    1

    需要特别说明，这里并不是真的在根目录下生成了bundle.js，而是在内存中生成了一个bundle.js文件。

6.webpack加载器

webpack默认只能打包js文件，非js文件webpack是处理不了的，需要调用loader加载器才能正常打包。loader加载器可以协助webpack打包特定的文件模块，比如：

  1. less-loader打包处理.less文件。
  2. sass-loader打包处理.sass文件。
  3. url-loader打包处理css中与url路径相关的文件。
