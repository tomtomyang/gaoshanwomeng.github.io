字符串模板引擎

ES5中的字符串缺乏多行字符串、字符串格式化、HTML转义等特性。

而ES6通过模板字面量的方式进行了填补，模板字面量试着跳出JS已有的字符串体系，通过一些全新的方法来解决问题。

1.基本用法

ES5字符串写法：

1let message = "我的宠物狗叫黑子，今年16岁了"

将其转化成ES6写法，其实非常简单：

只需把最外围的双引号（"）或者单引号（'） 转化成反引号（`）即可。

1let message = `我的宠物狗叫黑子，今年16岁了`

如果想在字符串内部使用反引号，只需使用反斜杠（ ）转义即可

1let message = `我的宠物狗叫\`黑子\`，今年16岁了`;
2console.log(message); // "我的宠物狗叫`黑子`，今年16岁了"

2.多行字符串

传统的JavaScript语言，输出模板通常是这样写的：

1var name = '黑子';
2var age = 8;
3$('#result').append(
4  '我的宠物狗叫 <b>' + name + '</b>\n' +
5      '今年\n' +
6      '<em>' + age+ '</em>岁，\n'+
7  '十分可爱!'
8);

但是在ES6中，要获得同样效果的多行字符串，只需使用如下代码：

1let name = '黑子';
2let age = 8;
3$('#result').append(
4  `我的宠物狗叫 <b>${name}</b>
5      今年 
6      <em>${age}</em>岁，
7  十分可爱!`
8);

对比两段拼接的代码，模板字符串使得我们不再需要反复使用双引号（或者单引号）了；而是改用反引号标识符

（`），插入变量的时候也不需要再使用加号（+）了，而是把变量放入${ }即可。

也不用再通过写 n 进行换行了，ES6 的模板字面量使多行字符串更易创建，因为它不需要特殊的语法，只需在想

要的位置直接换行即可，此处的换行会同步出现在结果中。简单、清晰、明了。

注意：如果使用模板字符串表示多行字符串，所有的空格和缩进都会被保留在输出之中。因此需要特别留意缩进。

1console.log(`我的宠物狗叫黑子
2                             今年16岁了`);
3结果：                             
4//我的宠物狗叫黑子
5//                             今年16岁了

以上代码中，模板字面量第二行前面的所有空白符都被视为字符串自身的一部分。

如果一定要通过适当的缩进来对齐文本，可以考虑在多行模板字面量的第一行空置并在后面的几行缩进

1let html = `
2<div>
3    <h1>Title</h1>
4</div>`.trim();

以上代码中，模板字面量的第一行没有任何文本，第二行才有内容。 HTML标签的缩进增强了可读性，之后再

调用trim()方法移除了起始的空行。

3.字符串中嵌入变量

1模板字面量看上去仅仅是普通JS字符串的升级版，但二者之间真正的区别在于模板字面量的变量占位符。

ES5写法：

1const age = 8;
2const message  = '我的宠物狗叫黑子，今年' + age*2 + '岁了' ;
3//我的宠物狗叫黑子，今年16岁了

ES6写法：

1const age = 8;
2const message  = `我的宠物狗叫黑子，今年 ${age*2} 岁了` ;
3//我的宠物狗叫黑子，今年16岁了

变量占位符允许将任何有效的JS表达式嵌入到模板字面量中，并将其结果输出为字符串的一部分。

如上面的例子，占位符 ${age} 会访问变量 age，并将其值插入到 message 字符串中。

既然占位符是JS表达式，还可以轻易嵌入运算符、函数调用等。

1const age = 8;
2const message = `我的宠物狗叫黑子，今年 ${(age*2).toFixed(2)} 岁了`;
3//"我的宠物狗叫黑子，今年 16.00 岁了"
4
5function fn() {
6  return "小黄";
7}
8`我朋友家的宠物叫${fn()}`
9//"我朋友家的宠物叫小黄"

4.带标签的模板字符串

要用 ES6 模板实现复杂一点的字符串处理逻辑，要依赖写函数来实现。

1幸运的是，除了在模板的插值表达式里想办法调用各种字符串转换的函数之外，ES6 还提供了更加优雅且更

容易复用的方案——带标签的模板字面量（tagged template literals，以下简称标签模板）

1标签模板的语法很简单，就是在模板字符串的起始反撇号前加上一个标签。
2
3let message = tag`黑子`;

在上面的代码中，tag就是模板标签。tag其实是一个函数，这个函数会被调用来处理这个模板字符串。

4.1 定义标签

 1let  name = '黑子',
 2     age = 8,
 3     message = tag`我的宠物狗叫${name}，今年${age}岁了`;
 4
 5function tag(stringArr, value1, value2) {
 6    console.log(stringArr);  //["我的宠物狗叫", "，今年", "岁了", raw: Array(3)]
 7                             //该数组有一个raw属性，保存的是转义后的原字符串
 8    console.log(value1);     // 黑子
 9    console.log(value2);     // 8
10    return;
11}

标签函数通常使用不定参数特性来定义占位符，从而简化数据处理的过程

1function tag(stringArr, ...values) {
2     console.log(values); //  ["黑子", 8]
3}

4.2 实际应用

“标签模板”的一个重要应用，就是过滤HTML字符串，防止用户输入恶意内容

 1var message =  filterHTML`<p>${sender} 你好啊</p>`;
 2function filterHTML(templateData) {
 3  var s = templateData[0];
 4  for (var i = 1; i < arguments.length; i++) {
 5    var arg = String(arguments[i]);
 6    // 转义占位符中的特殊字符。
 7    s += arg.replace(/&/g, "&amp;")
 8            .replace(/</g, "&lt;")
 9            .replace(/>/g, "&gt;");
10    // 不转义模板中的特殊字符。
11    s += templateData[i];
12  }
13  return s;
14}

上面代码中，sender变量往往是用户提供的，经过filterHTML函数处理，里面的特殊字符都会被转义。

1var sender = '<script>alert("买了个包")</script>'; // 恶意代码
2var message = filterHTML`<p>${sender} 你好啊</p>`;
3
4console.log(message);
5// <p>&lt;script&gt;alert("买了个包")&lt;/script&gt;  你好啊</p>

即使一个恶意命名的用户，例如黑客向其他用户发送一条骚扰信息，无论如何这条信息都会被转义为普通字符串，

其他用户不会受到潜在攻击的威胁。

标签模板的另一个应用，就是多语言转换（国际化处理）

 1let name = 'ES6专栏 ';  let  number = 666;  
 2let chinese = ['欢迎访问','您是第','位访问者'];
 3
 4i18n`Welcome to ${name}, you are the ${number}  visitor`;  
 5
 6function i18n(stringArr, ...values){
 7    let str='';  
 8    stringArr.map((item,index)=>{
 9        str += (chinese[index] +  (values[index] ? values[index] : '' ))   
10    });
11    console.log(str) 
12}
13//欢迎订阅ES6专栏 您是第666位访问者

5.新增的字符串方法

5.1 查找字符串

在以前在字符串中查找字符串的时候，都是使用indexOf方法。

ES6新增了三个方法来查找字符串 :

includes 方法会在给定文本存在于字符串中的任意位置时返回 true，否则返回 false 。

startsWith 方法会在给定文本出现在字符串开头时返回 true，否则返回 false 。

endsWith 方法会在给定文本出现在字符串末尾时返回 true，否则返回 false 。

1var msg = "Hello world!";
2
3console.log(msg.startsWith("Hello"));       // true
4console.log(msg.endsWith("!"));             // true
5console.log(msg.includes("d"));             // true
6
7console.log(msg.startsWith("d"));           // false
8console.log(msg.endsWith("world!"));        // true
9console.log(msg.includes("x"));             // false

每个方法都接收两个参数：需要搜索的文本和可选的起始索引值。

若第二个参数未提供，includes 和 startsWith 会从字符串的起始中开始检索，endsWith 则是从字符串的末尾。

实际上，第二个参数减少了需要检索的字符串的总量。

当提供第二个参数后，includes 和 startsWith 会以该索引为起始点进行匹配

1console.log(msg.includes("d", 8));          // true
2console.log(msg.includes("d", 11));         // false
3console.log(msg.startsWith("d", 10));       // true
4console.log(msg.startsWith("d", 9));        // false

endsWith 将字符串的长度与参数值相减并将得到的值作为检索的起始点

1console.log(msg.endsWith("d", 11));         // true ( 长度13 - 参数值11 = 起始点2 )
2console.log(msg.endsWith("d", 8));          // false ( 长度13 - 参数值8 = 起始点5 )

应用场景：

（1）利用includes检查用户使用的浏览器

1if (navigator.userAgent.includes('Chrome')) {
2     console.log("是谷歌浏览器");
3} else {
4      console.log("不是谷歌浏览器");
5}

（2）利用startsWith检查路径

1let urlstr = "http://www.baidu.com";
2let urlstr1 = "file:///C:/Users/xxx.html";
3console.log(urlstr.startsWith("https://"));    //false
4console.log(urlstr1.startsWith("file://"))     //true

（3）利用endsWith检查文件格式

1let path = "111.jpg";
2console.log(path.endsWith(".png"));   //false

5.2 字符串重复 – repeat方法

接受一个数字参数作为字符串的重复次数。该方法返回一个重复包含初始字符串的新字符串，重复次数等于参数。

例如：

1console.log("abc".repeat(4));       // "abcabcabcabc"
2let str = "小样儿";
3console.log(str.repeat(10));     //小样儿小样儿小样儿小样儿小样儿小样儿小样儿小样儿小样儿小样儿

5.3 填充字符串

string.padStart 往字符串前面填充（在字符串前面插入字符串）

string.padEnd 往字符串后面填充（在字符串后面插入字符串）

1let str = "Iphone";
2let leftStr = "我的";
3console.log(str.padStart(str.length + leftStr.length, leftStr));//我的Iphone
4
5let iphone = "Iphone";    
6let rightStr = "10";    
7console.log(iphone.padEnd(iphone.length + rightStr.length, rightStr));    //Iphone10
