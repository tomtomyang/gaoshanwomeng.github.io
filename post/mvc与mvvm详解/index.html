浅谈 MVC、MVP 和 MVVM 架构模式

架构模式：MVC与MVVM

结合vue、react、angular谈谈MVC、MVP、MVVM框架

1. 架构模式

MVC和MVVM都是架构模式！

  • MVC描述了「Model,View,Controller」三者之间的关系
  • MVVM描述了[Model,View,ViewModel(和Binder)」三者之间的关系

可以看出MVC和MVVM都是为了解耦「界面」和「数据」，只是解决方案不同！也可以说MVVM是MVC的一种变体，ViewModel+Binder可以说是Controller的一种实现方式。

为什么要解耦「界面」和「数据」？

早期的界面系统开发时，展示逻辑与数据是糅合在一起的！以早期的Java为例，所有的展示逻辑和业务逻辑都写在了Servlet/JSP中，导致了：

  • 单个文件的代码量较多，既包含了展示代码又包含了业务逻辑
  • 违背了职责单一性原则
  • 修改麻烦
  • 复用性差
  • 不易于维护
  • ……

2. MVC

1. 组成

MVC将系统拆分为控制器、视图和模型来解决上面的问题：

  • 控制器（Controller）

    执行业务逻辑

  • 视图（View）

    构建用户界面

  • 模型（Model）

    对数据进行保存

2. 优点

这样的拆分，提高了系统的：

  • 可扩展性：

    View、Model和Controller分别负责视图、数据和逻辑，可独立进化。能较方便的增加新功能。

  • 可维护性：

    结构清晰，多个View可以复用一个Model，减少了代码重复

  • 灵活性：

    Controller 可以用来连接不同的 Model 和 View 去完成用户的需求

3. 运作模式

用js来实现一个最最最简单的MVC架构就是：

 1/** 模拟 Model, View, Controller */
 2var M = {}, V = {}, C = {};
 3
 4/** Model 数据 */
 5M.data = "hello world";
 6
 7/** View 展示 */
 8V.render = (M) => { alert(M.data); }
 9
10/** Controller 逻辑 */
11C.handleOnload = () => { V.render(M); }
12
13/** 在页面加载的时候调用Controller*/
14window.onload = C.handleOnload;

  1. View 传送指令到 Controller
  2. Controller 完成业务逻辑后，要求 Model 改变状态
  3. Model 将新的数据发送到 View，用户得到反馈

4. React与MVC

虽然React一再强调自己只是一个构建界面的库，在宏观的角度，或者整个应用的角度来看，React只是充当了一个v的角色，但是当我们站在前端的角度思考问题，就会发现React组件内视图与数据的组织模式也是符合这种规范的。

在React组件内，我们要关注的是两个问题：

  1. 数据变了，视图如何变？
  2. 视图变了，数据如何变？

3. MVVM

MVVM是在MVC的基础上发展起来的，

1. 概念

View 与 Model 不发生联系，都通过ViewModel进行联络，也就是说提供一个中间层，实现数据驱动视图，视图修改数据，而视图和数据时不能直接进行对话的。

2. Vue与MVVM

宏观来说，Vue也只是一个构建视图的框架，那么在整个WEB应用中，只是负责页面的构建。但是我们把目光放小，站在前端的角度上，关注前端如何把数据与视图组织起来，以及数据和视图发生变化时的工作模式，这时候我们会发现，有另一层数据与逻辑的处理存在。

在Vue实例中，我们要关注的是两个问题：

  1. 数据变了，视图如何变？
  2. 视图变了，数据如何变？

先来分析第一个问题，数据变了，视图如何变？

发生这种情况，肯定是我们操作了Vue实例中的data数据，data数据发生改变，Vue实例中的数据监听机制就会被触发，监听到数据发生改变，更新页面。从这套流程中，我们可以分析出这么一套流程，Model->ViewModel->View。

再来分析第二个问题，视图变了，数据如何改变？

发生这种情况，肯定是用户进行了页面的操作，比如改变了input框中的值，Vue实例监听到了页面上发生了一些事件，那么就会去修改相应的data数据，这套流程中，我们可以分析出另一套流程，View->ViewModel->Model。

这里我们只分析Vue的工作流程，关于Vue是如何实现监听，实现数据双向绑定的，我们在另一篇文章中进行探讨《vue 数据双向绑定详解》

在这两个流程中我们分析出了，Vue的各个部分各自的职责，以及在MVVM中的身份：

  • View：最后生成的真实页面（HTML），这里要注意template是用来构建页面的模板，而不是最终真实的页面。
  • ViewModel：Vue实例，负责将视图和数据组织起来
  • Binder：Vue实例内部的观察者模式，双向数据绑定
  • Model：Vue实例中的data
