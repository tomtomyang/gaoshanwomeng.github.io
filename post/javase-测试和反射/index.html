反射

概念

  • 将类的各个部分封装为其它对象，这就是反射机制

好处

  • 解耦，增加程序的扩展性
  • 可以在程序运行过程中，操作这些对象

  • Class.forName(“全类名”);将字节码加载到内存，返回Class对象
    • 多用于配置文件，将类名配置在文件中
  • 类名.class:通过类名的属性classh获取
    • 多用于参数的传递
  • 对象.getClass()：getClass()方法在Object类中定义
    • 多用于对象获取字节码的方式

20 *            Constructor<?>[] getDeclaredConstructors()
21 *               返回 Constructor 对象的一个数组，这些对象反映此 Class 对象表示的类声明的所有构造方法。

23 *               返回一个 Constructor 对象，它反映此 Class 对象所表示的类的指定公共构造方法。
24 *            Constructor<?>[] getConstructors()
25 *              返回一个包含某些 Constructor 对象的数组，这些对象反映此 Class 对象所表示的类的所有公共构造方法。
26 *      3、获取成员方法
27 *            Field getDeclaredField(String name)
28 *              返回一个 Field 对象，该对象反映此 Class 对象所表示的类或接口的指定已声明字段。
29 *            Field[] getDeclaredFields()
30 *                返回 Field 对象的一个数组，这些对象反映此 Class 对象所表示的类或接口所声明的所有字段。
31 *             Method getMethod(String name, Class<?>... parameterTypes)
32 *              返回一个 Method 对象，它反映此 Class 对象所表示的类或接口的指定公共成员方法。
33 *           Method[] getMethods()
34 *              返回一个包含某些 Method 对象的数组，这些对象反映此 Class 对象所表示的类或接口
35 *              （包括那些由该类或接口声明的以及从超类和超接口继承的那些的类或接口）的公共 member 方法。
36 *      4、获取类名
37 *          String getName()
38 *           以 String 的形式返回此 Class 对象所表示的实体（类、接口、数组类、基本类型或 void）名称。

40public class Demo01Get {

43        Class personClass = Person.class;
44
45        //1、Field[] getFields() 获取所有的public修饰的成员变量
46        Field[] fields = personClass.getFields();
47        for (Field field : fields) {
48            System.out.println(field);
49        }
50        System.out.println("============");
51
52        //2、Field getField(String name)
53        Field a = personClass.getField("a");
54        //获取成员变量a的值
55        Person p = new Person();
56        Object value = a.get(p);
57        //设置a的值
58        a.set(p, "张三");
59        System.out.println(p);
60        System.out.println("==============");
61
62        //3、Field[] getDeclaredFields()
63        //获取所有的成员变量
64        Field[] declaredFields = personClass.getDeclaredFields();
65        for (Field field : declaredFields) {
66            System.out.println(field);
67        }
68
69        //4、Field getDeclaredField(String name)
70        Field d = personClass.getDeclaredField("d");
71        //因为d是私有变量，若要访问要忽略访问权限修饰符的安全检查
72        d.setAccessible(true);  //暴力fan'she
73        Object value2 = d.get(p);
74        System.out.println(value2);

注解简介

  • 注解实在jdk1.5之后的新特性
  • 说明程序的

本质

  • 注解本质就是一个接口，该接口默认继承Annotation接口

注解分类

  • 编写文档
    • 通过代码里标注的注解生成文档（生成doc文档）
  • 代码分析
    • 通过代码里标注的注解对代码进行分析（使用反射）
  • 编译检查
    • 通过代码里标注的注解让编译器实现基本的编译检查（override）

 5/**
 6 *   自定义注解
 7 *   格式：
 8 *      元注解
 9 *      public @interface 注解名称(){
10 *          属性列表（其实是抽象方法）
11 *      }
12 *   本质：
13 *      注解本质就是一个接口，该接口默认继承Annotation接口
14 *   属性：
15 *      接口中可以定义的抽象方法
16 *      要求：
17 *         1、属性的返回值类型
18 *             基本数据类型
19 *             String
20 *             枚举
21 *             注解
22 *             以上类型的数组
23 *         2、定义了属性，在使用时需要给属性赋值
24 *            在如果在定义属性时使用default关键字给属性默认初始化则使用注解时，可以不进行属性赋值
25 *            如果只有一个属性需要赋值且属性名为value，则value可以省略，直接赋值即可
26 *            数组赋值时，值使用大括号，若数组中只有一个元素，则大括号可以省略
27 *      元注解：
28 *          用来描述注解的注解
29 *          @target ：用来描述注解能够作用的位置
30 *          @Retention ：描述注解被保留的阶段
31 *          @Documented ：描述注解是否被抽取到api文档中
32 *          @Inherited ：描述注解是否被子类继承
33 */
34public @interface MyAnno {
35    int age();
36    //定义时默认赋值
37    String name() default "张三";
38
39    Person per();
40
41    MyAnno2 anno2();
42
43    String[] str();
44}

 55import java.io.BufferedWriter;
 56import java.io.FileWriter;
 57import java.io.IOException;
 58import java.lang.reflect.InvocationTargetException;
 59import java.lang.reflect.Method;
 60
 61/**
 62 *   简单的测试框架
 63 *    当主方法执行时，会检测所有加上Check注解的方法，判断是否有异常，记录到文件中
 64 */
 65public class TestCheck {
 66    public static void main(String[] args) throws IOException {
 67        //创建对象
 68        Calculator c = new Calculator();
 69        //获取文件字节码对象
 70        Class cls = c.getClass();
 71        //获取所有方法
 72        Method[] methods = cls.getMethods();
 73
 74        //创建存储异常文件
 75        BufferedWriter buw = new BufferedWriter(new FileWriter("bug.txt"));
 76
 77        //出现异常的次数
 78        int num = 0;
 79        for (Method method : methods) {
 80            //判断上述方法是否有Check注解
 81            if (method.isAnnotationPresent(Check.class)) {
 82                //有，则执行
 83                try {
 84                    method.invoke(c);
 85                } catch (Exception e) {
 86                    //捕获异常
 87                    num++;
 88                    buw.write(method.getName() + "方法出异常了");
 89                    buw.newLine();
 90                    buw.write("异常名称：" + e.getCause().getClass().getSimpleName());
 91                    buw.newLine();
 92                    buw.write("异常原因：" + e.getCause().getMessage());
 93                    buw.newLine();
 94                    buw.write("--------------");
 95                    buw.newLine();
 96                }
 97            }
 98        }
 99
100        buw.write("本次测试共出现" + num + "次异常");
101        buw.flush();
102        buw.close();
103    }
104}

小结

  • 以后大多数情况，我们仅仅会只用注解，而不去定义注解
  • 注解的使用者：编译器、解析程序
  • 注解不是程序的一部分，可以立即为注解就是一个标签
