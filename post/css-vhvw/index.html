1. vw vh 的介绍

vw,vh 是 css3 中给我们提供的新的单位，他们是相对于 viewport 视窗的宽高进行计算的单位。说白了，就是相对于浏览器的窗口的大小。

  • vw : 1vw 等于视口宽度的 1%
  • vh : 1vh 等于视口高度的 1%
  • vmin : 选取 vw 和 vh 中最小的那个
  • vmax : 选取 vw 和 vh 中最大的那个

2. vw、vh 与 百分比的区别

  1. % 是相对于父元素的大小设定的比率，vw、vh 是视窗大小决定的。
  2. vw、vh 优势在于能够直接获取高度，而用 % 在没有设置 body 高度的情况下，是无法正确获得可视区域的高度的。

3. vw、vh 与 px 的转换

3.1 直接计算

盒子高度 vh = 盒子高度 px / 总高度 px * 100vh

盒子宽度 vw = 盒子宽度 px / 总宽度 px * 100vw

移动端适配时一般都使用 vw，因为移动端上下滑动很方便，左右划带来糟糕的体验，因此尽量都让宽度上是适配的。

3.2 用 rem 转换

移动端的设计稿一般是 width：750px，那么存在一下关系：

  1. 100vw = 100% = 750px
  2. 1px = （100 / 750） vw = 0.13333vw
  3. 100px = 13.33vw

同时我们还要知道，1 rem=1 根组件 font-size 大小，如果我们让 根组件 font-size = 13.33vw，那么就得到一个新的关系：

  1. 1 rem = 13.33vw = 100px

那么如果我们知道了一个盒子的宽度为$x$px，那么这个盒子的宽度就可以写成$（x/100）$rem。

通过 rem 这个中间单位，我们就不用每次都换算 vw，只需要用 px 单位除 100，就可以了。

3.3 用 scss 函数来转换

 1$width: 100vw;
 2//px单位转换为vw
 3@function px2vw($size,$width:750px){
 4    @if(type-of($size)=="number" and unit($size)=="px"){
 5        @return $size*100vw/$width;
 6    }@else{
 7        @return $size;
 8    }
 9}
10#contact{
11    width: $width;
12    height: px2vw(1525.5px);
13}

 1//使用混合实现
 2.pxToVW (@px, @attr: width) {
 3    @vw: (@px / 750) * 100;
 4    @{attr}: ~"@{vw}vw";
 5}
 6
 7.box {
 8    .pxToVW(75);
 9    .pxToVW(150, height);
10}
