React Hooks 详解

1. Hooks 用途

对函数型组件进行增强，让函数型组件可以存储状态，可以拥有处理副作用的能力，让开发者在不使用类组件的情况下，实现相同的功能。

2. Hooks 解决的问题

Hooks的诞生，是为了对函数组件进行加强，使用函数组件解决类组件存在的问题，这些问题包括：

  1. 缺少逻辑复用的机制：

为了复用逻辑增加无实际渲染效果的组件，增加了组件层级，显示十分臃肿，增加了调试的难度以及运行效率的降低；

  1. 类组件经常会变得很复杂难以维护：

    将一组相干的业务逻辑拆分到了多个生命周期函数中，在一个生命周期函数内，存在多个不相干的业务逻辑；

  2. 类成员方法不能保证 this 指向的正确性；

关于类组件的一些缺点，在《react 函数组件详解》中有介绍。

3. 使用 Hooks

Hooks 意为钩子， React Hooks 就是一堆钩子函数， React 通过这些钩子函数对函数型组件进行增强，不同的钩子函数提供了不同的功能。

1. useState()

用于为函数组件引入状态：

 1import {useState} from 'react'
 2
 3function App() {
 4  const [count, setCount] = useState(0)
 5  return (
 6    <div>
 7      <span>{count}</span>
 8      <button onClick={()=>setCount(count+1)}> + 1</button>
 9    </div>
10  );
11}
12
13export default App;

接受唯一的参数即状态初始值，初始值可以是任意数据类型：

1const [count, setCount] = useState(0)
2const [person, setPerson] = useState({ name: '张三', age: 20 })

返回值为数组，数组中存储值和更改状态值的方法，方法名称约定以 set 开头，后面加上状态名称：

1const [count, setCount] = useState(0)

方法可以被调用多次，用以保存不同状态值：

1<button onClick={()=>setCount(count+1)}> + 1</button>
2<button onClick={()=>setPerson({name: '李四', age: 30})}>setPerson</button>
3<button onClick={()=>setPerson({...person, name: '李四'})}>setPerson</button>

参数可以是一个函数，函数返回什么，初始状态值就是什么，函数只会被调用一次，在初始值是动态值的情况：

1// 当初始值是动态值
2// 这样写每次渲染都会执行
3const propsCount = props.count || 0
4// const [count, setCount] = useState(propsCount)
5
6// 应该这样写
7const [count, setCount] = useState(() => {x
8  return props.count || 0 // 只有第一次执行的时候才会执行
9})

设置状态值方法的参数可以是一个值也可以是一个函数，设置状态值方法的方法本身是异步的，如果代码依赖状态值，那要写在回调函数中：

1function handleCount () {
2  setCount((count) => {
3    const newCount = count + 1x
4    document.title = newCount
5    return newCount
6  })
7}
8
9<button onClick={handleCount}> + 1</button>

2. useReducer()

useReducer 是另一种让函数组件保存状态的方式，可以将 dispatch 传给子组件使用

 1import { useReducer } from "react";
 2
 3export default function App () {
 4
 5  function reducer (state, action) {
 6    switch (action.type) {
 7      case 'increment':
 8        return state + 1
 9      case 'decrement':
10        return state - 1
11      default:
12        return state;
13    }
14  }
15
16  const [count, dispatch] = useReducer(reducer, 0)
17  return (
18    <div>x
19      <button onClick={() => dispatch({type: 'decrement'})}>-1</button>
20      <span>{count}</span>
21      <button onClick={() => dispatch({type: 'increment'})}>+1</button>
22    </div>
23  )
24}

4. useEffect()

让函数型组件拥有处理副作用的能力，类似生命周期函数

  1. useEffect 执行机制

可以把 useEffect 看做 componentDidMount, componentDidUpdate 和 componentWillUnmount 这三个函数的组合

useEffect(() => {}) => componentDidMount, componentDidUpdate useEffect(() => {}, []) => componentDidMount useEffect(() => () => {}) => componentDidUpdate, componentWillUnmount useEffect(() => () => {}, []) => componentWillUnmount

 1import { useEffect, useState } from "react";
 2import ReactDOM from 'react-dom'
 3
 4export default function App () {
 5
 6  const [count, setCount] = useState(0)
 7
 8  // 组件挂载完成之后执行，组件数据更新之后执行
 9  // useEffect(() => {
10  //   console.log('123')
11  // })
12
13  // 组件挂载完成之后执行
14  // useEffect(() => {
15  //   console.log('456')
16  // }, [])
17
18  useEffect(() => {
19    return () => {
20      console.log('组件被卸载了')
21    }
22  })
23
24  return (
25    <div>x
26    <span>{count}</span>
27    <button onClick={() => setCount(count+1)}>+1</button>
28    <button onClick={() => ReactDOM.unmountComponentAtNode(document.getElementById('root'))}>卸载组件</button>
29    </div>
30  )
31}
  1. useEffect 使用方法

(1). 为 window 对象添加滚动事件 (2). 设置定时器让 count 数值每隔一秒增加 1

 1import { useEffect, useState } from "react";
 2import ReactDOM from 'react-dom'
 3
 4export default function App () {
 5
 6  const [count, setCount] = useState(0)
 7  function onScroll () {
 8    console.log('页面滚动了')
 9  }
10
11  useEffect(() => {
12    window.addEventListener('scroll', onScroll)
13    return () => {
14      window.removeEventListener('scroll', onScroll)
15    }
16  }, [])
17
18  useEffect(() => {
19    const timerId = setInterval(() => {
20      setCount(count => {
21        const newCount = count + 1
22        document.title = newCount
23        return newCount
24      })
25    }, 1000);
26    return () => {
27      clearTimeout(timerId)
28    }
29  }, [])
30
31  return (
32    <div>
33      <span>{count}</span>
34      <button onClick={() => { ReactDOM.unmountComponentAtNode(document.getElementById('root')) }} >卸载组件</button>
35    </div>
36  )x
37}
  1. useEffect 解决的问题 (1). 按照用途将代码进行分类（将一组相同的业务逻辑归置到了同一个副作用函数中） (2). 简化重复代码，是组件内部代码更加清晰
  2. 只有指定数据发生变化时触发 effect
 1import { useEffect, useState } from "react";
 2
 3export default function App () {
 4
 5  const [count, setCount] = useState(0)
 6  const [person, setPerson] = useState({name: '张三'})
 7
 8  useEffect(() => {
 9    // person 的变化不会触发 useEffect , 因为第二个数组参数中只监听了 count
10    document.title = count
11    console.log(111)
12  }, [count])
13
14  return (
15    <div>
16      <span>{count}</span>
17      <button onClick={() => setCount(count + 1)}> + 1</button>
18      <button onClick={() => setPerson({name: '李四'})}>setPerson</button>
19    </div>
20  )
21}
  1. useEffect 钩子函数结合异步函数 useEffect 中的参数函数不能是异步函数，因为 useEffect 函数要返回清理资源的函数，如果是异步函数就变成了返回 Promise. 可以写成自执行函数的形式：
1useEffect(() => {
2    (async () => {
3      await axios.get()
4    })()
5  }, [])

5. useMemo()

useMemo 的行为类似 Vue 中的计算属性，可以检测某个值的变化，根据变化只计算新值。 useMemo 会缓存计算结果，如果检测子没有发生变化，及时组建重新渲染，也不会重新计算，此行为可以有助于避免在每个渲染上进行昂贵的计算。

 1import { useState, useMemo } from "react";
 2
 3export default function App () {
 4
 5  const [count, setCount] = useState(0)
 6  const [bool, setBool] = useState(true)
 7  const result = useMemo(() => {
 8    console.log('111') // 只有 count 改变才会重新执行这个回调函数
 9    return count * 2
10  }, [count])
11
12  return (
13    <div>
14      <span>{count} {result}</span>
15      <span>{bool ? '真' : '假'}</span>
16      <button onClick={() => setCount(count + 1)}> + 1</button>
17      <button onClick={() => setBool(!bool)}>setBool</button>
18    </div>
19  )
20}

2
3const Foo = memo(function Foo () {
4  return <div>I am Foo</div>

7. useCallback()

性能优化，缓存函数，使用组件重新渲染时得到相同的函数实例。否则每次父组件渲染，函数变量的实例都会变化，导致里层组件被重新渲染

 1import React, { memo, useState, useCallback } from 'react';
 2
 3function App () {
 4  const [count, setCount] = useState(0);
 5  const resetCount = useCallback(() => setCount(0), [setCount])
 6  return (
 7    <div>
 8      <span>{count}</span>
 9      <button onClick={() => setCount(count + 1)}>+1</button>
10      <Counter resetCount={resetCount}/>
11    </div>
12  )
13}
14
15const Counter = memo (function Counter () {
16  console.log('Foo')
17  return(
18    <div>
19      <button onClick={props.resetCount}>resetCount</button>
20    </div>
21  )
22})
23
24export default App

8. useRef()

获取DOM元素对象

 1import React, { useREf } from 'react';
 2
 3function App () {
 4  const box = useRef();
 5
 6  return (
 7    <div ref={box}>
 8      <button onClick={() => console.log(box)}>获取div</button>
 9    </div>
10  )
11}

使用 useRef 钩子函数保存数据(跨组件周期)

即使组件重新渲染，保存的数据仍然还在，保存的数据被更改不会触发组件重新渲染。

 1import { useRef, useState, useEffect} from "react";
 2
 3export default function App () {
 4  const [count, setCount] = useState(0)
 5  let timeId = useRef() // 夸组件生命周期
 6  useEffect(() => {
 7    // 使用这个 ref 的 current 属性存储数据
 8    timeId.current = setInterval(() => {
 9      setCount(count => count + 1)
10    }, 1000);
11  }, [])
12
13  const stopCount = () => {
14    console.log(timeId.current)
15    clearInterval(timeId.current)
16  }
17
18  const box = useRef()
19  return (
20    <div ref={box}>
21      <span>{count}</span>
22      <button onClick={() => stopCount()}> 停止 </button>
23    </div>
24  )
25}

9. 自定义 Hook

  • 自定义hook 是标准的封装和共享逻辑的方式
  • 自定义hook 是一个函数，以use 开头
  • 自定义hook 其实就是逻辑和内置hook 的组合

如何使用自定义 Hook:

 1import React, { useEffect, useState } from 'react';
 2import axios from 'axios';
 3
 4function useGetPost () {
 5  const [post, setPost] = useState({});
 6  useEffect(() => {
 7    axios.get('https://jsonplaceholder.typicode.com/posts/1').then(response => setPost(response.data))
 8  }, [])
 9  return [post, setPost]
10}
11
12function App () {
13  const [post, setPost] = useGetPost()
14  return (
15    <div>
16      <div>{post.title}</div>
17      <div>{post.body}</div>
18    </div>
19  )
20}
21
22export default App
