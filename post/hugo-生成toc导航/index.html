hugo 生成toc导航
📅 2020-06-05 👦 Tomtom Young 📖 1436字 ⏱ 3分钟

1.hugo自带toc模版

hugo本身带有生成文章导航的功能，使用{{ .TableOfContents}}变量，会在页面中渲染出id为TableOfContents的nav样式。

1.使用方法

官方文档给出的使用模版是：

1{{ if and (gt .WordCount 400 ) (.Params.toc) }}
2<aside>
3    <header>
4    <h2>{{.Title}}</h2>
5    </header>
6    {{.TableOfContents}}
7</aside>
8{{ end }}

这样就会自动查找markdown文档生成的h标题。

2.缺点

使用hugo默认的{{.TableOfContents}}有如下几个缺点：

  1. 从h1开始查找。我的文章一般都是标题为h2，文章内小标题从h3开始，这样生成的toc会有两段很大的缩进，不够智能，样式也很难看。
  2. 锚点指向出错。以本地调试为例，我的文章页面应该是localhost/post/文章名/，点击锚点指向的应该是localhost/post/文章名/#锚点/，但是这个默认生成的却是指向了localhost/#锚点，因此不能实现跳转。
  3. 不能平滑滚动。即使锚点指向的是正确地址，跳转的时候是直接生硬的跳转，而不是平滑滚动。
  4. 不能滚动监听。只能跳转和显示。

2.自定义toc模版

1.新建文件

在主题文件夹/layouts/partials/新建toc.html，位置可以不同，只要在引入的时候写对就可以

2.编辑toc.html

在toc.html里加入以下内容：

 1<div class="post-toc">
 2    <div class="toc-header">CATALOG</div>
 3    {{- $headers := findRE "<h[2-4].*?>(.|\n])+?</h[2-4]>" .Content -}}
 4    {{ .Scratch.Set "last_level" -1 }}
 5    {{/* at least one header to link to */}}
 6    {{- $has_headers := ge (len $headers) 1 -}}
 7    {{- if $has_headers -}}
 8        {{- range $headers -}}
 9            {{- $last_level := $.Scratch.Get "last_level" -}}
10            {{- $header := . -}}
11            {{- $base := ($.Page.File.LogicalName) -}}
12            {{- $anchorId := ($header | plainify | htmlUnescape | anchorize) -}}
13            {{- $href := delimit (slice $base $anchorId) "#" | string -}}
14            {{- range findRE "[2-4]" . 1 -}}
15                {{- $next_level := (int .) -}}
16                {{- if eq $last_level -1 -}}
17                    <ul class="toc-h{{ . }}">
18                {{- else if gt $next_level $last_level -}}
19                    {{- range seq (add $last_level 1) $next_level}}
20                        <ul class="toc-h{{ . }}">
21                    {{- end -}}
22                {{- else if lt $next_level $last_level -}}
23                    {{- range seq (add $next_level 1) $last_level}}
24                        </ul>
25                    {{- end -}}
26                {{- end -}}
27                <li><a href="{{ relref $.Page $href }}" class="toc-link">{{- $header | plainify | htmlUnescape -}}</a></li>
28                {{ $.Scratch.Set "last_level" $next_level }}
29            {{- end -}}
30        {{- end -}}
31    {{- end -}}
32</div>
33

这个模版是从h2开始查找，而且如果没有h2，目录自动从h3开始生成，不会有那么多缩进造成不美观。

3.在single.html引入模版

single.html是文章的展示页面，可能会有不同，只要是文章的展示页面就可以。

1  {{ if .Site.Params.toc | default false }}
2  {{ partial "taxonomy/toc.html" . }}
3  {{ end }}

{{ partial “taxonomy/toc.html” . }}根据模版路径更改。

4.编辑config.toml

在config.toml中加入：

1[params]
2toc = true

这样就可在页面中显示toc了。

3.添加平滑滚动效果

打开页面的js文件，新定义一个平滑滚动函数：

1function tocScroll() {
2    var target = decodeURI(this.hash.replace(/^#/, ''));
3    $('html,body').animate({scrollTop: $(":header[id='" + target + "']").offset().top - 70}, 300);
4}

这里面的target是为了处理中文标题，offset().top后面的70自己根据顶部导航栏的大小修改，如果顶部没有导航栏就可以不用写，300是滑动的时间，单位是毫秒。

给链接绑定平滑滚动效果：

1var tocLinks = document.querySelectorAll('.toc-link');
2    for( var i in tocLinks){
3        tocLinks[i].onclick = tocScroll;
4    }

4.添加滚动监听效果

滚动监听功能会用到bootstrap的scrollspy插件，页面滚动到章节内容时，toc目录中对应的条目就会变成active状态，自定义li.active的样式就能清楚的看出当前章节的位置。 添加滑动监听有两种方法，一是在所要监听的元素上设置data-spy和data-target属性，通过data-target属性值将toc目录组件与被监听元素绑定起来。二是在js文件中绑定这些元素属性。个人觉得在html中绑定，不是很灵活，而且不好找到元素，所以选择在js中绑定。代码如下. 其中offset依旧是顶部导航栏的高度, 这里多加的5px，是为了调整点击跳转和页面平滑滚两种情况下目录条目active状态的些许误差。这个值根据实际页面情况调整，并非固定值。

1$('body').scrollspy({ target: "#page-scrollspy", offset: navbarHeight+5 });
